Autolayout -from Training to leading

There are more methods here ,very similar to use .Which method we should use to change and then achieve the frame We want..Then you end up with one single one line code to try ,but finally you got angry as Nothing work rightly though you called these methods.

So all the way out is much a huge black box right for many developers never knows what’s happening well.The good news is this set needsDisplay method in the lower corner left is not related to layout at all ,so we should get rid of it .

So our questions arises,what’s the layout ????

????????????????

1,Layout is that :
What to draw ,where to draw,It does not define how to draw as that’s drawing algorithm ,it’s a completely different thing.

For instance as the pic

If we just have one element here ,we define its X,Y,width and height.That’s work well..

X+Y = Origin,width+ height = size ,so The frame is Origin + Size.

So put it simple layout is another word to determine frames for the views on screen.

2,Determine frames for all views on Screen

How we do that?

2.1The easy way to do it is to write down the frame explicitly right .(Manually portion).Traditional approach how it was done several years ago .We just write these code as below:

Button.frame = CGRect(x:0,y:0,width:34,height:34)

But this approach worked great as Several years ago we just had particular devices size for iPhone.Just 320 pixels width.

But now there are more devices here ,for instance iPhone7 plus ,iPhone 7,iPhone6 plus and so on .So when we have more devices ,this apporach doesnot work well any more.This is because this is how we want our button to resize right actually we wanna have horizital padding to screen on the both side.
