Why write Unit test ?

1,Tests Reduce Bugs in New Features

We advocate writing tests as you write new code. Tests do not eliminate bugs, but they dramatically reduce the number of bugs as you add new features.

2,Test reduce Bugs in existing Features

With well-tested code, introducing new features rarely breaks existing functionality. If a new feature breaks existing functionality, existing tests fail immediately, allowing you to pinpoint the problem and fix it. Without the tests, you may introduce a bug that is not found for days or weeks.

3,Test is a good documenataion

A concise code example is better than many paragraphs of documentation.If we read the documentation ,we always move forward and backward between different files.

4,Test is so fun

If you wanna challenge yourself,writing test will be so fun …

5,Test improve your design

Writing tests forces you to make your code testable. Specifically, you tend to rely less on dubious patterns like singletons and global variables, instead making your classes loosely-coupled and easier to use. Code that is tightly-coupled or requires complex initialization is hard to test.

6,Test make development faster

Test can reduce the bugs,as we all know ,we spend most time on debugging application .10% of time to write code ,when your code became more and more ,bugs are tough to debug ,so it will slow down your speed to develop .

7,Test can force yourself to slow down and think

There are one common mistake developers always make- received task and then begin to write code at once ……Stupid behaviors to do ,it makes yourself pain.

8,Test reduce fear.

If you never write Test,How to know your Application work well .So making you far away fear,today start to write Unit test .

9,Test defend other coders

At least you are on top of themselves ,that’s your special skill.Compared with them you own more skills .

10.Test allows you to refactory

With tests, you are more able to change code throughout the lifetime of an application. Tests provide a safety net, allowing you to refactor at any time without fear of breaking existing code, so you can constantly improve the design of your program.
