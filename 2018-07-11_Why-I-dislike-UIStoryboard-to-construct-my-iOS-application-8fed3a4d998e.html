Why I dislike UIStoryboard to construct my iOS application

Nowadays Storyboard is becoming fashion in the field of iOS development.As one iOS programmer I dislike it .

1,Easy to commit comflict with other people

When your team is large,less than five people ,it’s not a great way to develop your iOS application

2,The code cannot be reused during the following process

When you wanna make your code reused ,you should say goodbye to Storyboard.As it wrote less code ,but when you met complex interface ,it’s tough to meet your requirement.

3,Easy to meet some weird issues

Today I use the Storyboard,but the weird issue always displayed.Sometimes you cannot connect outlet to the related class file.The Xcode helps you to modify your constraints and then let your constraint fuzzy.Sometimes it can make your interface black

4,Storyboard can lead you lack of more knowledge .

As we all know ,Storyboard is suitable for the junior iOS engineer ,as your knowledge is limited,use it ,you can develop fast ,but if you wanna be an advanced iOS engineer,from now on ,stop using it and write it via programming code.

5,Storyboard is so slow.

When use the storyboard ,everything was so slow,you must improve your mac hardware,but use code to do it ,so fast .....

From now on ,I will say goodbye to Storyboard as It’s so stupid.Compared with Android development ,xml to write the UI layout code and then the code of activity became less,so why not take advantage of it for apple?
